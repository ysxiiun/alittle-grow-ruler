# 孕期体重拆分数据日期与记录时间

> 序号：002
> 日期：2026-04-12 18:28
> 任务类型：新功能

## 任务概述
为孕期体重模板引入独立的数据日期和记录时间，避免凌晨录入的睡前体重被统计到次日。统计分析按数据日期聚合，记录列表保留完整记录时间用于感知实际录入时刻。

## 技术方案摘要
在 `data_entries` 增加 `data_date` 字段并兼容回填历史数据，前端孕期体重表单拆分“数据日期”和“记录时间”，凌晨 `0:00-3:00` 的睡前体重默认回退前一天，分析与导入统一按 `data_date` 计算。

## 遇到的问题与解决方案
切回远端仓库基线后本地依赖缺失，补执行 `npm install` 后重新运行 `npm run build` 完成验证。

## 技术决策记录
将“凌晨睡前归前一天”的业务规则限定在孕期体重模板，其他模板仍沿用原有单时间模型，降低影响面。
